DESIGN REQUIREMENTS

 

The conceptual design is to solve the problems identified. The problem identified and the corresponding design requirements are outlined in the following table.

The problems Design requirements
The color is distracting/not emphasizing on the important items/confusions for green red color blindness. Using a set of icons representing completed and incompplete instead of using big color area.
The presentation of multiple majors is so easy to be ignored. List different major buttons on the top separately, using selected/unselected states to distinguish which major the user is viewing.
Display of information is not very organized. Reorganized the page, show the summary of each single major and shorten each items under requirement check (hover to expand all items but indicates whether is completed or not)
Not sure whether graduation has already been applied or not. Show "apply for graduation" if is not applied while show "check application status" when already applied.
Not showing whether graduation has already been applied or not. Once go to application page, show application status (applied or nor applied).
Not showing the application summary/users need to go to application to check. Show application status on the application main page.
Not giving feedback after actions taken such as withdraw, submit or modify application etc. Give feedback for each action taken.
Layout of POS form is not very organized. Reorganized the page, categorized information based on different topics.
Not giving feedback after actions taken such as save, validate or submit form etc. Give feedback to each action taken.
Not clear showing whether the form has been approved or not. Show the summary of approval (or not) at the top.
